Meta(FaceBook) releases Memlab; A JavaScript framework that finds memory leaks

By Christopher Hernandez

While researching how to better optimize, measure, and control page load and interaction time, along with the size of JS code. Previously they had systems to alerted them when problems arose both before and after code was in production. They had never done much in terms of managing memory while in browser, but discovered Facebook's memory usage and memory crashes on the client side were rising. Obviously high memory usage would negatively affect loading and interaction, and by proxy user engagement metrics like how many people were using it and for how long.
