Meta(FaceBook)™ releases Memlab; A JavaScript Framework that Finds Memory Leaks

By Christopher Hernandez

While researching how to better optimize, measure, and control page load and interaction time, along with the size of JS code. Previously they had systems to alerted them when problems arose both before and after code was in production. They had never done much in terms of managing memory while in browser, but discovered Facebook's memory usage and memory crashes on the client side were rising.

Obviously, high memory usage would negatively affect parts of the user experience and website statistics such as

  • Load time
  • Interaction length
  • Number of active users
  • Time spent on websites
  • number of activities performed
